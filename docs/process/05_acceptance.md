# 05_acceptance.md — 驗收鏈與 QA 反轉規則

> **★★2026-07-09 流程改（用戶定案）——本檔 R1-R6 三層驗收鏈在 user-in-loop 模式下降級為「藍圖按需調用的稽核能力」，不再是 per-slice 硬閘**：
> - **release-pass 權 → 藍圖**：量測員標準 full_probe 床（`03b_measurer.md §④`）出完整數字 → 藍圖判、有問題升用戶。正式 QA release-gate stage 砍。
> - 三層機器（矛盾偵測/常駐漏斗/戲感審計）**續存為工具**——藍圖/系統做深度稽核時調用；只是不再是「每次交用戶前必全綠」的閘。
> - **綁 user-in-loop**：用戶深度在線=事實驗收權威。**哪天轉自動交付（用戶不看迴圈）→ 本檔三層硬閘全數回歸**。
> - `escaped_defects` ledger（R6 系族）**續記**。

> 源事故：用戶多次被迫當 QA（抗議後制度又繞回）。藍圖 `2026-07-04-qa-inversion` 反轉原則,本檔=規則本體（系統 own）。
> **精神：教訓變機器,不變文件。**每條規則附 enforcement,無一句「下次注意」。

## R1 原則反轉（總綱）

**用戶眼球=願景/品味輸入,永遠不是驗收工具。**
驗收鏈三層,**全綠才交付用戶**：

| 層 | 內容 | 機器形式 |
|---|---|---|
| ① 矛盾偵測（充足性,2026-07-04 判準修正） | **可解釋性非量級**：每個率（含 0）必須能從世界狀態解釋。合理的 0=健康;**矛盾（想要+可行+沒發生）=病**。assert=「矛盾=0」,非量地板（量配額=逼子系統表演=功能模擬器化,廢） | seeded bed 矛盾率 FAIL 條件 |
| ② 常駐漏斗 | 行為鏈分階段轉化率,常駐 bed 輸出（非一次性探針） | 回歸可見的率表 |
| ③ 戲感審計 | 跑 default,讀 ticker 流+截圖,以觀者身分出審計報告 | ticker-dump + 世界句子審計（見 R4） |

**判決三類**（取代 荒謬/稀薄/健康 量級檔）：`可解釋`（健康,含合理的0,附解釋句）/ `矛盾`（斷鏈,修這種）/ `未知`（查不出因=觀測缺,補探針非直接修）。

「閉環/done」宣告權=①②③全綠+**驗收官（QA）判決**（`04_qa.md` 四職,2026-07-04 復活升級）。**藍圖與系統均無權憑機制修復宣告——系統不自判自己蓋的世界。**

## R2 escaped-defect 制

用戶發現的任何問題=**逃逸缺陷=流程事故**。流程：
1. 立檔 `docs/escaped_defects.md`（ledger,常駐）。
2. 判定哪層該抓而沒抓。
3. 補那層機器（機器補上=該筆結案,缺陷本體修復另計）。

## R3 率不是計數

戲類指標一律帶分母：成交/訂單、成交/月/村對、delivered/dispatched、同化/俘獲…。
**enforcement**：harness/bed 輸出格式強制帶率;spec 驗收寫裸計數=打回。

## R4 世界句子審計（③的核心動作）

把統計唸成世界的句子——「十幾個村想買賣幾百次、成交六次」「獸隊在市場收購武器」——問句=「**說得通說不通**」（非「多不多」）。說不通=矛盾或未知。
**enforcement**：交付用戶前,**QA session** 跑/讀 default seed ticker dump+截圖,出觀者報告（怪點列表或「無怪點」聲明）。系統備機器,QA 出判決。跳過=流程違規。

## R5 反確認偏誤

支持「已修好/該閉了」結論的數字,**強制做除法+對脈絡**——越順眼越要拆（claim-time trigger 延伸）。
enforcement：handback 驗收段引用數字時必附分母與脈絡句。

## R7 修理協議（防「專搞一環弄壞別的」,2026-07-04 判準修正）

1. **治矛盾不追配額**：修=接通斷鏈（想要的能發生）;**禁**=灌權重讓數字達標（扭曲全世界人格/稀缺邏輯）。
2. **修一環、全環對照**：任何單環修 → 率表**全環回歸**（seeded diff+全率表前後對照）。目標環修好但他環變形=**fail**。QA 判決看整張表,不看單列。

## R6 用戶抗議=流程規則候選

用戶對流程的抗議/預警,預設升級為 process 規則提案（進本檔或對應 process doc）,**不當單次交付註記消化**。

## 試點綁定

首個試點=貿易環點火軌（spec `2026-07-04-trade-loop-ignition-design.md` Task 3）：done=修好+三層機器存在並綠。之後各維度驗收照此升級,含**回頭補已「過」的維度**（結盟弧 delivered=0 同病,戲存在性時代的「✓」逐個重驗充足性）。
